Chrome console: “message channel closed”

If you see Uncaught (in promise) Error: A listener indicated an asynchronous response by returning true, but the message channel closed before a response was received (or runtime.lastError with the same text), it almost always comes from a browser extension, not from the Doubow web app.

Extensions that inject scripts sometimes tell Chrome they will reply asynchronously to a message, then never call the reply callback or the tab navigates away first. That is a bug or race in the extension, not something Doubow can fix in application JavaScript.

What to do

  1. Incognito with extensions disabled (or a fresh profile). If the message disappears, confirm it is an extension.
  2. Disable extensions one by one (ad blockers, grammar tools, wallet extensions, “enhancer” scripts, etc.) until the noise stops.
  3. If the app behaves correctly, you can ignore this console line for production debugging of Doubow itself.

Doubow does not use chrome.runtime message channels in the shipped web app; there is nothing to patch on our side for this specific error.
