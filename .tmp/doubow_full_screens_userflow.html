The Human-in-the-Loop guarantee — every outbound message must be explicitly approved. This is enforced at the database level, not just in the UI.
Application state machine — 7 states
DISCOVERED
Job scraped, stored in catalog
→
SCORING
Scorer agent calculating fit
→
DRAFTING
Drafter agent writing outreach
→
PENDING_APPROVAL
Waiting for user review
User approves
APPROVED
DB written by user action ONLY
Sender agent triggered
403 if not APPROVED in DB
SUBMITTED
sent_at recorded
User edits
edited_by_user = true
Diff stored in llm_logs
Returns to PENDING_APPROVAL
Re-review prompt
User rejects
REJECTED
Pipeline ends, archived
System error
FAILED
Alert + llm_log entry
RETRY (max 3×)
Exponential backoff

API enforcement — the trust guarantee
POST /applications/{id}/submit called
Any source: UI or direct API call
→
DB status check
SELECT status FROM applications WHERE id=?
→
Status ≠ APPROVED
HTTP 403 Forbidden returned
Status = APPROVED → Sender queued → message dispatched → SUBMITTED
The only path to SUBMITTED requires a prior human write to APPROVED in the database
