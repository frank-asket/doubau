Doubow 4-week sprint plan for West Africa v1 launch

Week 1 Foundation + first adapter Days 1–7

Decide your coverage honesty strategy today

Your UI must tell users "strong coverage: remote/global + some Nigeria, Ghana, Senegal local roles." Don't promise what Adzuna + RemoteOK can't deliver. Write this copy before any code.

Provider ToS + attribution checklist (Week 1, non-negotiable)

For each provider: required UI labels, required link-outs, caching rules, and any restrictions. Bake into acceptance criteria: every job card shows provider label + "view original" (and uses the provider job URL where required).

Build ProviderAdapter base class

fetch() → raw, to_canonical(raw) → JobIn, source_label, source_region. 30–50 lines. Everything else inherits this.

Remote OK adapter (first quick win)

Public JSON feed. Strong remote coverage for diaspora + global roles. Map to canonical and persist with source label "Remote OK" + required link-out URL.

Adzuna adapter — Nigeria/Ghana queries (add breadth)

Hit Adzuna with queries scoped to country=ng and country=gh (and optional keyword sets per persona). Map to canonical. Source label: "Adzuna".

Fingerprint dedup in Redis

hash(title + company + location), 48h TTL. Runs before embed call — prevents wasting OpenAI tokens on duplicates.

Embed + write to pgvector

Embed f"{title} at {company} — {description[:400]}". Pin text-embedding-3-small. Store vector + all canonical fields. Never change the model without full re-index.

Gate: 200+ jobs indexed, attribution + link-out shown on every card, one manual match test passing
Week 2 Resume pipeline + matcher Days 8–14

Resume parser

Accept PDF + DOCX. Extract: title, skills list, location, years of experience, seniority. Use Claude API (structured output) for speed — but never block matching: if parsing fails, store raw extracted text and proceed with embeddings-only matching.

Embed resume profile

Embed f"{title}, {', '.join(skills)}, {location}, {seniority}" with the same text-embedding-3-small model. Same template as job embedding = better cosine alignment.

ANN query + re-ranker

Pull top 50 via pgvector cosine. Re-rank with weighted score: vector sim (0.5) + location match (0.2) + seniority match (0.2) + recency (0.1). Return top 15 to user.

Location matching for West Africa

Normalise location strings: "Lagos", "Lagos, NG", "Nigeria" all → NG. Remote jobs get a soft boost for all West Africa users. Don't filter out remote roles because they lack a GH/NG tag.

Source attribution on every result

Every job card: "via Adzuna · posted 3 days ago · [view original]". Link must go to the actual apply URL. Non-negotiable for ToS and user trust.

Gate: upload a real resume, get ≥10 relevant results in <3s
Week 3 Index prefill + internal testing Days 15–21

Run ingestion daily via Celery beat

Target 2,000+ indexed jobs before any user sees the product. Schedule: Adzuna at 06:00 UTC, RemoteOK at 07:00 UTC. Monitor index size daily.

Test with 10 real team resumes

Manually inspect top 20 results per resume. If <3 of 10 get strong matches → your re-rank weights or query template is wrong. Fix before week 4. Don't skip this.

Add job freshness expiry

Mark stale = true on jobs older than 30 days. Exclude from default queries. If provider lacks a posted timestamp, fall back to ingested_at for recency. Show "posted/ingested N days ago" on every card — stale jobs destroy trust fast.

Instrument match quality metrics

Log: jobs indexed, match latency p50/p95, avg results per resume, click-through on apply links, and a lightweight satisfaction signal (hide / thumbs down) to tune ranking without waiting for apply clicks.

Gate: 2,000+ jobs indexed, 8/10 internal resumes get strong matches
Week 4 Enroll first users Days 22–28

Only open if week 3 gate passes

If internal testing is weak, delay enrollment by one week and fix the matcher. One bad first impression from a known contact is worse than a one-week delay.

Invite 20–30 known users (warm network only)

Ghana/Nigeria diaspora + local professionals from your own network. Not a public launch. You need qualitative feedback, not volume. Ask them to talk to you after upload.

Collect structured feedback per user

3 questions: (1) Were results relevant? (2) Any roles you expected but didn't see? (3) Did you click apply on anything? Answers directly drive week 5 priorities.

Watch for coverage gaps by persona

Early-career + remote: likely fine. Mid-career diaspora: depends on seniority filter. Local hire in Accra/Lagos: expect thin results — be honest about this in the UI, don't hide it.

Gate: >60% of beta users click apply on at least one result
